# Vibe 体系未来演进规划 (Further Plan)

## ✅ 已完成阶段：可视化官网页面 (Vibe Gallery)
**状态：已上线/已完成核心开发**
Markdown 文件对 AI 来说是完美的，但对人类来说是灾难。我们已经完成了基础的“AI 资产调色盘”：
- **视觉优先**：支持 UI 风格与 Motion 动效的分类展示。
- **直观预览**：UI 风格支持跳转原站，Motion 动效支持视频/GIF 弹窗预览。
- **一键 Copy to Cursor**：点击直接复制出一段包裹好的 Markdown/YAML 提示词，供 AI 消费。

## 🚀 首要任务 (Next Priority)：乐高式拼装 (The Vibe-Mixer)

**为什么这是首要任务？**
UI 风格（皮相）和 交互动效（骨肉）现在被分成了两个库，这在架构上很完美。但在实际使用场景中，用户往往需要**同时**告诉 AI：“我要这个长相，加上这个动效”。

**具体执行路径：**
1. **交互设计**：在网站上提供一个类似“购物车”或“混合器”的悬浮面板。
2. **用户操作**：用户可以勾选 `1个 UI 风格` + `1~N个 交互动效`。
3. **一键融合**：网站前端通过脚本，将选中的多个 Markdown 文件的内容进行智能拼接，生成一份 **“缝合版”的超级 Prompt/Skill**。
4. **AI 消费**：AI 最擅长处理这种融合设定的需求，用户只需粘贴一次，AI 就能同时掌握皮相与骨肉。

## 🔮 终极杀器：开发 MCP Server (降维打击)

**概念**：为 Vibe 体系开发一个官方的 **MCP Server (Model Context Protocol)**。
**体验颠覆**：
现在大家还在手动复制 `.md` 文件。如果有了 MCP：
- 用户在 Cursor 里挂载 `vibe-mcp-server`。
- 用户直接在聊天框说：“我要做一个登录页，请从 MCP 里读取 `Stripe风格` 和 `弹性抽屉` 参数”。
- **AI 会直接通过 MCP 协议向你的云端/本地库请求最新的 UI Token 和物理参数**。
- 你的库有任何更新，所有用户实时生效，彻底免去复制粘贴的繁琐！

## 🛡️ 持续护城河：AI 避坑防偏指南 (Anti-Patterns)

大模型在生成代码时，经常会犯一些“通病”。我们需要在每个风格的 `Don'ts`（禁忌）里不断积累“驯服 AI 的经验”：
- *“AI 总是喜欢给暗黑模式加纯白色的阴影”* -> 规范里硬性规定：`不要使用带亮度的阴影，使用 1px rgba(255,255,255,0.1) 的边框替代`。
- *“AI 总是把动效写成 1 秒钟那么长，显得很卡”* -> 规范里硬性写死：`Duration 严禁超过 300ms`。

**这会让 Vibe 库不仅是一个“设计规范”，更是一个“驯服 AI 的说明书”，这才是其他传统设计规范库所不具备的核心壁垒。**
